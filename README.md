# vDisk USB Bootloader Setup · U 盘维护与启动盘制作工具

本仓库提供 **USB BOOTLoader Setup**（`usb_rom_vDisk.au3` 及配套资源）：在 Windows 上一键 **清理、分区并写入** 可启动 U 盘，并从您指定的 **Boot Server** 拉取引导内核、维护组件与镜像相关资源。它面向机房、教室与分支办公等场景，把「做一张能在现场救急、又能回连中心服务」的维护盘，从反复手工拷贝，收敛成 **选盘 → 填服务器地址 → 制作** 的几步操作。

我们相信，**好的基础设施工具应当配得上清晰的说明与可复现的构建方式**。将工程放在 GitHub，是为了让交付同事、渠道与集成商能 **同步同一套脚本与资源**；一线运维按 README 即可理解 **跨网段场景下该填哪个 IP、会下载哪些文件、失败时看哪份日志**，而不是依赖口耳相传的私盘版本。

---

## 为什么在跨 VLAN、跨网段环境里它特别有用

很多单位的 **办公网、教室网、机房网** 分属不同 VLAN 或物理网段，终端侧不一定能直接访问镜像库或管理控制台，但通常仍能访问 **某台可达的 Boot / 维护服务器**（或由您通过路由、ACL 放通 **HTTP 7012** 一类端口到固定主机）。

此时不必把整包镜像拷进 U 盘再在各地分发——只需在 **任意一台能访问该服务器的 Windows 电脑** 上运行本工具：

- 填入 **Boot Server IP**（对制作机而言「路由可达」即可，不必与目标终端同网段）。
- 工具会通过 HTTP 从服务器拉取 `corex64.gz`、`vmlinuzx64`、`rom/disk.zip`、`rom/diskgpt.zip` 等组件并完成写入。

也就是说：**制作端与业务 VLAN 解耦**；只要维护人员笔记本能 ping 通或访问到 Boot Server，就能为 **另一个网段里即将维护的机器** 预先准备好启动盘。现场再用该 U 盘完成 **下载 / 上传镜像、维护、恢复** 等操作（具体能力以您环境中所部署的 vDisk / 维护体系为准）。

## 能做什么（能力概览）

| 方向 | 说明 |
|------|------|
| **全新制作** | **Clean And Writeing**：格式化 U 盘、建立分区布局、写入 PE / 引导文件，并从服务器下载引导与 ROM 资源。 |
| **增量更新** | **Update BOOTLoader**：在已有分区结构上更新引导与 ROM 包，适合服务端升级后快速刷新 U 盘。 |
| **启动方式** | 制作结果 **同时支持传统 BIOS 与 UEFI（EFI）启动**（与常见机房维护需求一致）。 |
| **参数与备注** | **Comment** 对应启动参数侧需求（可与手册中「启动参数」说明对齐）；**Boot Server IP** 会写入使用习惯（注册表 `HKEY_CURRENT_USER\Software\osvusbbootloader`）。 |

## 使用步骤（简版）

1. **以管理员身份运行** 编译后的程序（源码中 `#RequireAdmin`，需提升权限才能正确访问磁盘与分区工具）。
2. **Select Device**：在下拉列表中选择目标 U 盘。**务必核对盘符与容量，选错可能导致数据丢失。**
3. **Boot Server IP**：填写 Boot 服务器地址（工具通过 `http://<IP>:7012/...` 下载资源，请确保网络与防火墙放通）。
4. **Comment**：按需填写启动相关说明或参数（可与现场规范一致）。
5. 点击 **Clean And Writeing** 开始完整制作，或 **Update BOOTLoader** 仅更新引导与 ROM 包。
6. 若下载失败，请查看临时目录下的 **`wget_download.log`**（路径随 `%TEMP%\boot` 变化），便于排查连通性与证书等问题。

## 服务端与网络要求（实施侧 checklist）

- 制作机到 Boot Server **TCP 7012** 需可达（工具使用 HTTP 拉取资源）。
- 典型下载路径包括（节选）：`:7012/corex64.gz`、`:7012/vmlinuzx64`、`:7012/rom/disk.zip`、`:7012/rom/diskgpt.zip`（以您实际服务端为准）。
- **跨 VLAN 场景**：在三层设备上为目标网段配置到 Boot Server 的路由或策略放行即可；**制作工具只要求「运行工具的电脑能访问该 IP:7012」**。

## 源码与本地构建

- **语言**：AutoIt 3（`usb_rom_vDisk.au3`）。
- **嵌入资源**（与脚本同目录，供 `FileInstall` 使用）：`wget.exe`、`bootia32.efi`、`bootx64.efi`、`bin.zip`、`7z.exe`、`7z.dll`、`pe.fba` 等；另有 `bootloader.lst` 等配置参与打包流程。
- 使用 **AutoIt + SciTE** 或您的打包脚本编译为独立 EXE 即可分发；具体编译选项见脚本头部 `#PRE_*` 注释。

依赖命令行与分区工具（由 `bin.zip` 释放），部分安全软件可能误报；若为企业内网环境，建议将工具路径加入白名单（与用户手册中说明一致）。
